# <a name="what-is-blockchain-data-manager-for-azure-blockchain-service"></a>Azure 区块链服务的区块链数据管理器是什么？

区块链数据管理器可以捕获、转换 Azure 区块链服务事务数据并将其传送到 Azure 事件网格主题，提供与 Azure 服务的可靠、可缩放区块链账本集成。

在大多数企业区块链方案中，区块链账本是解决方案的一个组成部分。 例如，若要将资产从一个实体转移到另一个实体，需要使用一个机制来提交事务。 然后，需要使用一个机制来读取账本数据，以确保事务发生且被接受，并将生成的状态更改集成到端到端解决方案。 在此示例中，如果你要编写一个用于转移资产的智能合同，可以使用区块链数据管理器来集成链外应用程序和数据存储。 在资产转移示例中，如果资产在区块链中转移，则区块链数据管理器会通过事件网格传送事件和属性状态更改。 然后，可以使用事件网格的多个可能的事件处理程序在链外存储区块链数据，或者实时对状态更改做出反应。

区块链数据管理器执行三项主要功能：捕获、转换和传送。

![区块链数据管理器的功能](./media/data-manager/functions.png)

## <a name="capture"></a>捕获

每个区块链数据管理器实例连接到一个 Azure 区块链服务成员事务节点。 只有有权访问该事务节点的用户才能创建连接，确保对客户数据进行适当的访问控制。 区块链数据管理器实例从事务节点可靠捕获所有原始块和原始事务数据，并可以扩展以支持企业工作负荷。

## <a name="transform"></a>转换

可以通过在区块链数据管理器中配置智能合同应用程序，使用区块链数据管理器来解码事件和属性状态。 若要添加智能合同，请提供合同 ABI 和字节码。 区块链数据管理器使用智能合同项目来解码和发现合同地址。 将区块链应用程序添加到实例后，当智能合同已部署到联盟并向配置的目标发送解码的事件和属性状态时，区块链数据管理器会动态发现智能合同地址。

## <a name="deliver"></a>传送

区块链数据管理器支持对任意给定的区块链数据管理器实例使用多个事件网格主题出站连接。 可将区块链数据发送到一个或多个目标。 使用区块链数据管理器可为任何区块链部署构建基于事件的可缩放数据发布解决方案。

## <a name="configuration-options"></a>配置选项

可根据解决方案的需求配置区块链数据管理器。 例如，可以：

* 为某个 Azure 区块链服务成员预配单个区块链数据管理器实例。
* 为每个 Azure 区块链服务事务节点预配一个区块链数据管理器实例。 例如，专用事务节点可以使用自身的区块链数据管理器实例来保持机密性。
* 一个区块链数据管理器实例可以支持多个输出连接。 可以使用一个区块链数据管理器实例来管理某个 Azure 区块链服务成员的所有数据发布集成点。
