# <a name="module-twin-json-schema-azure-event-grid"></a>Azure 事件网格 (的模块克隆 JSON 架构) 

IoT Edge 上的事件网格与 IoT Edge 生态系统集成，并支持通过模块克隆创建主题和订阅。 它还将报告所有主题和事件订阅的当前状态报告给模块克隆上的报告属性。

> [!WARNING]
> 由于 IoT Edge 生态系统中的限制，以下 json 示例中的所有数组元素都编码为 json 字符串。 请 `EventSubscription.Filter.EventTypes` 参阅 `EventSubscription.Filter.AdvancedFilters` 以下示例中的和键。

## <a name="desired-properties-json"></a>所需属性 JSON

* 在创建主题时，主题部分中每个键值对的值都具有与用于 API 的相同的 JSON 架构 `Topic.Properties` 。
* 在创建主题时， **EventSubscriptions** 部分中每个键值对的值都具有与用于 API 的相同的 json 架构 `EventSubscription.Properties` 。
* 若要删除某个主题，请将其值设置为 `null` 所需的属性。
* 不支持通过所需属性删除事件订阅。

```json
{
    "topics": {
        "twinTopic1": {},
        "twinTopic2": {
            "inputSchema": "customEventSchema"
        }
    },
    "eventSubscriptions": {
        "twinTopic1WebhookSub": {
            "topic": "twinTopic1",
            "retryPolicy": {
                "eventExpiryInMinutes": 120,
                "maxDeliveryAttempts": 30
            },
            "destination": {
                "endpointType": "WebHook",
                "properties": {
                    "endpointUrl": "https://localhost:4438"
                }
            },
            "filter": {
                "subjectBeginsWith": "^",
                "subjectEndsWith": "$",
                "isSubjectCaseSensitive": false,
                "includedEventTypes": "[\"et1\",\"et2\"]",
                "advancedFilters": "[{\"value\":true,\"operatorType\":\"BoolEquals\",\"key\":\"data.b\"},{\"values\":[\"\\\"\",\"c\"],    \"operatorType\":\"StringContains\",\"key\":\"data.s\"}]"
            }
        },
        "twinTopic2EdgeHubSub": {
            "topic": "twinTopic2",
            "deliveryPolicy": {
                "approxBatchSizeInBytes": 200000,
                "maxEventsPerBatch": 25
            },
            "destination": {
                "endpointType": "EdgeHub",
                "properties": {
                    "outputName": "twinTopic2EdgeHubSub"
                }
            },
            "filter": {
                "advancedFilters": "[{\"value\":true,\"operatorType\":\"BoolEquals\",\"key\":\"dAt\\\"A.a\"},{\"values\":[\"\\\"\",    \"c\"],\"operatorType\":\"StringContains\",\"key\":\"dAt\\\"A.a\"}]"
            }
        }
    }
}
```

## <a name="reported-properties-json"></a>报告的属性 JSON

模块克隆的报告属性部分包括下列信息：

* 模块的存储区中存在的主题和订阅集
* 创建所需的主题/事件订阅时遇到的任何错误
* 任何启动错误 (如所需属性 JSON 分析失败)
